## Manual Action Matrix

| Action | Current behavior | Safety boundary |
| --- | --- | --- |
| Attach Existing | Checks reachable admin API and records attached copy in UI. | Does not stop or take ownership of the external runtime. |
| Start Proxy | Spawns `codex-helper serve --codex --host 127.0.0.1 --port <port> --no-tui --desktop-managed` via `CODEX_HELPER_CLI_PATH`/sibling CLI lookup and polls admin API. | Marks runtime as desktop-managed through existing CLI owner marker path; tray hardening remains TDC-090. |
| Stop Owned | Requires exact `STOP OWNED PROXY`. | Only accepted for desktop-owned runtime; calls runtime shutdown endpoint. |
| Remote Stop Attached | Requires exact `STOP ATTACHED PROXY`. | Separate warning button/copy; if attached runtime cannot shutdown, command returns detach-only semantics. |
| Codex Switch On | Requires exact `SWITCH CODEX` from Settings. | Uses non-interactive core switch API; Dashboard only points user to Settings confirmation. |
| Codex Switch Off | Requires exact `SWITCH OFF CODEX` from Settings. | Restores through core switch state path. |
| Runtime reload / probe / balance refresh | Exposed as explicit page actions. | Disabled when live admin state is not usable. |
| Provider/global/session overrides | Exposed in Providers/Settings advanced areas. | Uses typed Tauri commands and admin API mutation endpoints; no secret material is displayed. |
