# 2026-05-31 - RBR-010 Docker Gate Repair

With Docker Desktop Linux daemon running, the first image build exposed two defects:

- `cargo chef prepare --bin codex-helper-server` omitted the workspace root manifest from the recipe, so `cargo chef cook` could not find `Cargo.toml`;
- `ENTRYPOINT ["/usr/bin/tini", "--"]` made `docker run image --help` ask tini to execute `--help`.

Fixes:

- prepare the full workspace recipe, then cook only `codex-helper-server`;
- make the image entrypoint execute `codex-helper-server` by default;
- override Compose entrypoint only for the one-time `/data/config.toml` initialization shell.

Evidence is recorded in `EVIDENCE_AND_GATES.md`.
