# 2026-05-22 — TDRP-070 provider common edit parity

## Scope

- Add common provider edit forms for safe single-endpoint providers.
- Patch config TOML without dropping advanced provider fields.
- Keep multi-endpoint and route-graph-heavy provider editing on the raw TOML path.

## Changes

- Added `save_common_provider` in `apps/desktop/src-tauri/src/commands/providers.rs`.
- Registered the command in the Tauri invoke handler.
- Added temp-file and pure TOML patch tests for:
  - single-provider common edits;
  - advanced-field preservation;
  - multi-endpoint rejection without overwriting;
  - non-http(s) URL rejection.
- Extended provider card view mapping with editable provider metadata.
- Added inline Provider card edit form for alias, base URL, enabled state, and optional env var names.
- Added Zod validation for provider common edits.
- Added frontend tests for safe single-endpoint editing and multi-endpoint raw TOML fallback.
- Added `ResizeObserver` test mock because Radix Switch needs it under jsdom.
