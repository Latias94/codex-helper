# 2026-05-22 — TDRP-060 signing, installer, and update posture

## Scope

- Decide whether the first Tauri replacement release can safely enable automatic updates.
- Document signing, installer, release channel, updater prerequisites, and deferral.
- Make Settings UI honest while updater signing/artifact hosting is not ready.

## Decision

Do not enable `tauri-plugin-updater` for the first replacement release yet.

The first replacement release policy is:

- publish Windows NSIS installer artifacts through GitHub Releases;
- keep automatic update checks disabled in the app;
- enable updater only after signing keypair, CI private-key secret, embedded public key, HTTPS update endpoint, updater artifacts, and rollback operations are all real and smoked.
