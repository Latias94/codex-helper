# 2026-05-22 — TDRP-080 packaged lifecycle smoke partial automation

## Scope

- Start TDRP-080 packaged smoke in an isolated environment.
- Avoid touching the user's active local helper runtime.
- Capture the first repeatable evidence for the installed Windows app.

## Changes

- Added `scripts/tdrp_080_packaged_smoke.ps1`.
- The script installs the Windows NSIS artifact into a temporary directory.
- It isolates state via `CODEX_HELPER_HOME` and `CODEX_HELPER_DESKTOP_ADMIN_URL=http://127.0.0.1:6211`.
- It clears `CODEX_HELPER_CLI_PATH` and legacy `CODEX_HELPER_CLI` so packaged smoke does not depend on developer CLI overrides.
- It uses Win32 window enumeration and `WM_CLOSE` instead of broad UIA scans.

## Verification

Command:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File docs/workstreams/tauri-desktop-replacement-parity/scripts/tdrp_080_packaged_smoke.ps1
```

Result: PASS for the automated subset.

Passed checks:

- NSIS installer ran with exit code 0.
- Temporary install directory contained `codex-helper-desktop.exe`.
- Temporary install directory contained bundled `codex-helper.exe`.
- Packaged app started and displayed the main native window.
- Native close hid the main window while leaving the desktop process alive.
- Second launch focused/restored the existing packaged instance.

## Concerns / follow-up

- TDRP-080 remains incomplete.
- Broad Windows UI Automation scans against WebView/system tray can hang on this machine, so future automation should avoid deep desktop tree scans.
- Still needed:
