# 2026-05-22 — TDRP-050 launch at login

## Scope

- Replace the inert Settings "开机启动" row with a real launch-at-login control.
- Use a Tauri-supported OS integration instead of a mock setting.
- Avoid touching the developer machine's active codex-helper runtime.

## Changes

- Added `tauri-plugin-autostart` through `pnpm tauri add autostart`.
- Registered the autostart plugin in `apps/desktop/src-tauri/src/lib.rs`.
- Added `@tauri-apps/plugin-autostart` wrappers in `apps/desktop/src/lib/tauri/commands.ts`.
- Added `useLaunchAtLogin` in `apps/desktop/src/features/settings/hooks.ts`.
- Updated Settings:
  - launch-at-login is a real switch backed by the plugin;
  - tray remains shown as enabled/read-only;
  - startup-time proxy auto-start remains disabled with explicit conservative copy.
- Extended the shared `Switch` component so rows can pass accessibility labels, disabled state, and change handlers.
- Added a frontend test proving the launch-at-login switch calls the autostart guest binding.
