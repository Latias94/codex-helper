# CDC-060 Journal

Date: 2026-05-26
Owner: codex

## Summary

Clarified official OpenAI identity vs relay continuity and surfaced explicit continuity-domain guidance in Codex relay diagnostics.

## Changes

- Added `expected.continuity` to the Codex capability profile.
- Updated the remote compaction v1 reason to state that OpenAI identity chooses the `/responses/compact` path but does not prove relay encrypted-state sharing.
- Added top-level relay capability `continuity` diagnostics:
  - selected continuity-domain key;
  - whether the domain is explicit;
  - configured endpoint count;
  - same-domain endpoint count;
  - whether state-bound failover is allowed inside the domain;
  - operator warnings and recommendations.
- Documented `continuity_domain` guidance for relay chains and direct OpenAI setups in `docs/CONFIGURATION.md` and `docs/CONFIGURATION.zh.md`.
- Updated evidence payload test fixtures for the new diagnostics field.

## Notes

Same host, same base URL, same provider brand, and same relay domain remain diagnostic context only. Explicit `continuity_domain` is the only configured proof that multiple provider endpoints intentionally share encrypted response state.
