# CDC-040 Journal - Soft Affinity Escape

Date: 2026-05-26

## Summary

CDC-040 is complete.

Ordinary conversation turns now use soft session affinity even when the route graph is configured with `Hard` affinity. The selected behavior is:

- Provider-state-bound compact keeps the configured/hard selector.
- Ordinary turns prefer the prior session endpoint.
- Ordinary turns escape to another healthy endpoint when the pinned endpoint is unavailable.

## Changes

- Added soft affinity selection methods to `RoutePlanExecutor`.
- Kept the default `select_supported_*_with_runtime_state` semantics configured/hard for state-bound callers.
- Updated provider execution to choose the soft selector only when `RequestContinuityDecision` is not provider-state-bound.
- Added a regression for ordinary `/v1/responses` where session affinity points to provider `b`, provider `b` is marked `BreakerOpen`, provider `c` is healthy, and the second ordinary turn succeeds through `c` instead of returning false 502.
- Added a routing unit test proving the soft selector can escape configured `Hard` affinity while existing hard selector behavior remains intact.
