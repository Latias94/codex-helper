# CDC-050 Journal - Explicit Continuity Domains

Date: 2026-05-26

## Summary

CDC-050 is complete.

The proxy now has an explicit continuity-domain model:

- Provider config and endpoint config can set `continuity_domain`.
- Endpoint `continuity_domain` overrides provider `continuity_domain`.
- If no explicit domain is configured, the effective domain is the provider endpoint itself.
- Same host, same base URL, or same DNS domain is never used as automatic proof of encrypted-state continuity.
- State-bound provider failover is allowed only after route selection is restricted to an explicit shared continuity domain.

## Changes

- Added `continuity_domain` to `ProviderConfigV4` and `ProviderEndpointV4`.
- Added `ContinuityDomainKey` and continuity-domain carrying runtime identities.
- Added candidate continuity-domain propagation in `routing_ir`.
- Added route-attempt state restriction to keep state-bound fallback inside the known continuity domain.
- Allowed provider failover for state-bound requests only inside explicit continuity domains.
- Tightened runtime upstream identity migration so continuity-domain changes reset retained state.
- Preserved existing registry/API behavior by carrying existing domain metadata forward and not auto-inlining default endpoints with endpoint-level domain metadata.
