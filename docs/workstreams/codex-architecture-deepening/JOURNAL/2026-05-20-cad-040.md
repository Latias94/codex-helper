# 2026-05-20 — CAD-040 Relay Diagnostic Case Registry

## Scope

- `crates/core/src/proxy/codex_relay_probe.rs`
- `crates/core/src/proxy/codex_relay_capabilities.rs`
- `crates/core/src/proxy/codex_relay_live_smoke.rs`
- workstream docs under `docs/workstreams/codex-architecture-deepening/`

## What changed

- Added a `CodexRelayProbeCase` registry that owns the existing probe wire contracts:
  - `model_catalog` -> `GET /models` read-only
  - `responses` -> `POST /responses` validation-only `{}`
  - `remote_compaction_v1` -> `POST /responses/compact` validation-only `{}`
- Reworked capability diagnostics to iterate registered probe cases, then project observations back to the existing response shape.
- Added a `CodexRelayLiveSmokeCaseDescriptor` registry for compact, hosted image generation, and Responses WebSocket smoke cases.
- Split live-smoke execution into descriptor-driven HTTP and WebSocket paths.
- Moved default-case selection, acknowledgement requirement, explicit-only warnings, request body builders, and classifiers behind descriptors without changing public payload shape.

## Behavior preserved

- No `/responses/compact` fallback was added.
- Capability response fields remain `observed.models`, `observed.responses`, and `observed.responses_compact`.
- Live-smoke defaults still run only `responses_compact`.
- Hosted image generation and Responses WebSocket smoke remain explicit-only.
- Existing acknowledgement token is still required before any live upstream smoke I/O.
- Evidence appenders continue serializing the same response payloads.
