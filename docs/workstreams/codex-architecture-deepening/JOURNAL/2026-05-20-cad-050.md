# 2026-05-20 — CAD-050 Proxy Integration Test Harness

## Scope

- `crates/core/src/proxy/tests/harness.rs`
- `crates/core/src/proxy/tests/mod.rs`
- `crates/core/src/proxy/tests/failover/response_semantics.rs`
- workstream docs under `docs/workstreams/codex-architecture-deepening/`

## What changed

- Added a focused proxy integration-test harness with:
  - `TestProxyServer` and `TestUpstreamServer` wrappers that abort server tasks on drop;
  - `spawn_test_upstream`, `spawn_test_proxy`, and `spawn_proxy_service`;
  - default `upstream_config` from a spawned upstream;
  - `post_responses_json` and `post_compact_json` request helpers;
  - `find_finished_request` for polling request-state evidence.
- Migrated the first response-semantics slice, covering:
  - compact path forwarding;
  - zstd request normalization/passthrough/rejection;
  - compact unsupported status diagnostics;
  - non-retryable 400 and non-retryable client-error failover behavior;
  - retry-enabled 404 behavior;
  - model-support filtering;
  - model mapping.

## Behavior preserved

- Assertions for status, body shape, hit counts, upstream body bytes, content encoding, model mapping, and request-state evidence remain in the test bodies.
- The harness avoids opaque scenario builders; it only removes repeated lifecycle/default-config/request boilerplate.
- No production code behavior changed in CAD-050.
