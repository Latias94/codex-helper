## What changed

- Added `CodexPatchPlan` as the pure switch-on policy seam.
- Moved public patch mode/options definitions into `codex_patch_plan` and re-exported them from `codex_integration` for compatibility.
- Centralized provider identity and provider TOML bool decisions:
  - `name = "codex-helper"` vs `name = "OpenAI"`;
  - `requires_openai_auth`;
  - `supports_websockets`.
- Centralized auth patch strategy:
  - restore original auth if helper wrote a facade;
  - patch complete ChatGPT bridge auth;
  - patch empty imagegen facade auth.
- Centralized effect ordering:
  - restore modes write `config -> auth -> switch state`;
  - auth-facade modes write `switch state -> config -> auth`.
- Reworked switch-on adapters:
  - TOML writes use `switch_on_codex_toml_with_plan`;
  - auth decisions use `auth_edit_for_switch_on_plan`;
  - filesystem side effects use `apply_switch_on_effects`.
- Connected capability profile expected-shape calculation to the patch-plan seam so diagnostics and switch-on use the same policy source.

## Behavior preserved

- Preset alias parsing remains covered by config-storage tests.
- `responses_websocket = true` remains valid only for official relay/imagegen presets.
- ChatGPT bridge still validates Codex auth.json rather than helper upstream auth.
- Imagegen and official-imagegen still require helper upstream runtime readiness and still write the empty auth facade.
- Official relay still restores any helper-written auth facade and does not synthesize a compact fallback.
- Remote-control enablement and diagnostics were not changed.
